SpatiaLite supports

a Geometry data

type conformant to the international standard OGC-SFS (Open Geospatial Consortium - Simple Feature SQL).

"http://www.opengeospatial.org/standards/sfs">http://www.opengeospatial.org/standards/sfs


Geometry is an

abstract data type with seven related

concrete sub-classes.
You cannot directly instantiate a

"Courier New, monospace">Geometry (because this

one is and abstract class, and doesn't corresponds to any

actual implementation): but you can freely instantiate any

related sub-class.

  • "Courier New, monospace">GEOMETRYCOLLECTION isn't

  • And this sub-class isn't generally supported by ordinary GIS sw

    (viewers and so on).

So it's very rarely used in the real GIS professional world.

 

WKT and WKT

notations

Geometry is a

very complex data type: accordingly to this, OGC-SFS defines

two alternative standard notations allowing to represent Geometry

values:

  • the WKT (Well Known Text) notation is intended to

    be user friendly (not really so user friendly

    after all, but at least human readable).

  • the WKB (Well Known Binary) notation on the other

    side is more intended for precise and accurate

    import/export/exchange of Geometries between

    different platforms.

Please note well: both WKT and WKB notations are intended to support standard data exchange

(import/export);
anyway the actual format internally used by SpatiaLite is a

different one, i.e. BLOB Geometry.
You must never be concerned about such internal

format:
you simply have to use the appropriate conversion functions so to

convert back and forth in standard

"Courier New, monospace">WKT or

"Courier New, monospace">WKB.

  • the Hex() function is a standard SQL function allowing to represent binary

    values as hexadecimal encoded text strings.

  • the Spatial SQL function

    "Courier New, monospace">ST_GeomFromText() converts

    any valid WKT expression into an internal

    "Courier New, monospace">BLOB Geometry value.

  • "Courier New, monospace">ST_GeomFromWKB() converts

    any valid WKB expression into an internal

    "Courier New, monospace">BLOB Geometry value.

  • ST_AsText() converts an internal

    "Courier New, monospace">BLOB Geometry value into the

    corresponding WKT expression.

  • ST_AsBinary() converts an internal

    "Courier New, monospace">BLOB Geometry value into the

    corresponding WKB expression.

 

SELECT

ST_GeometryType(ST_GeomFromText('POINT(1.2345

2.3456)'));

"Courier New, monospace">POINT
SELECT

ST_GeometryType(ST_GeomFromText('POINTZ(1.2345 2.3456

10)'));
POINT

Z
SELECT

ST_GeometryType(ST_GeomFromText('POINT ZM(1.2345 2.3456 10

20)'));
POINT

ZM
ST_GeometryType() will return the Geometry Type from the given internal BLOB Geometry value.

  • Please note: when using not-2D dimensions,

    declaring e.g. '

    "Courier New, monospace">POINTZ' or '

    "Courier New, monospace">POINT Z' is absolutely the

    same: SpatiaLite understands both notations indifferently.

 

SELECT

ST_Srid(ST_GeomFromText('POINT(1.2345 2.3456)'));

"Courier New, monospace">-1
SELECT

ST_Srid(ST_GeomFromText('POINT(1.2345 2.3456)',

4326));

"Courier New, monospace">4326
ST_Srid() will

return the SRID from the given internal

"Courier New, monospace">BLOB Geometry> value.

  • Please note: both

    "Courier New, monospace">ST_GeomFromText() and

    ST_GeomFromWKB() accept an optional

    "Courier New, monospace">SRID argument.
    If the SRID is

    unspecified (not at all a good practice), then

    "Courier New, monospace">-1 is assumed.
